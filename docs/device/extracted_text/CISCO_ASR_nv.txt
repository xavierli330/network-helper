install nv satellite
To download and activate the software image on the satellite, use the install nv satellite command in EXEC
mode.
install nv satellite {satellite idall}{transfer | activate | update{file file name | reference name of
reference}}
Syntax Description
Specifies the unique identifier of the satellite on which the image must be transferred.
satellite id
Performs the operation on all currently active satellites that are not already at the target
version.
all
Downloads the image from the host to the satellite device.
transfer
Performs the install operation on the satellite.
activate
Performs the transfer and activate of SMUs/base package on the satellite in a single
operation on the satellite.
update
Performs the update operation on the specified file or list of files
file
Performs the update operation on the specified reference
reference
Specifies the file or the list of files on which the operation is to be performed.
file name
Specifies the name of a previously created reference on which the operation is to be
performed.
reference name
Command Default
No default behavior or values
Command Modes
EXEC mode
Command History
Modification
Release
This command was introduced.
Release 4.2.1
The update option was introduced in this command.
Release 6.3.1
Usage Guidelines
This command only works on satellites that are fully connected and authenticated.
Note
If the activate keyword is run directly, then the software image is transferred to the satellite and also activated.
Note

minimum preferred links
To configure the minimum number of preferred satellite fabric bundle-ether links in a dual home satellite
network topology, use the minimum preferred links command in the satellite nV fabric link redundancy
configuration mode.
minimum preferred links number
Syntax Description
Specifies the number of minimum number of active links to the host. The range is 1 to 64.
number
Command Default
If you do not enable this parameter on any host, then it is assumed as 0 by default on that host.
Command Modes
Satellite Fabric Link Redundancy
Command History
Modification
Release
This command was introduced.
Release
5.3.1
Usage Guidelines
In order to configure minimum preferred links, the interface must be a bundle-ether interface. If you do not
enable this parameter on any host, then it is assumed as 0 by default on that host. Hence, the decision of
failover is judged using that value.
You can either go to the redundancy mode to configure minimum preferred links or you can specify
redundancy keyword before configuring the minimum preferred links.
Note
Task ID
Operation
Task ID
read,
write
ethernet-services
Example
This example shows how to execute the minimum preferred links command:
RP/0/RSP0/CPU0:router # configure
RP/0/RSP0/CPU0:router(config)# interface Bundle-Ether 1
RP/0/RSP0/CPU0:router(config-int)# nV satellite-fabric-link satelite 200
RP/0/RSP0/CPU0:router(config-satellite-fabric-link)# redundancy minimum preferred links 2
(OR)
RP/0/RSP0/CPU0:router(config-satellite-fabric-link)# redundancy
RP/0/RSP0/CPU0:router(config-nV-red)# minimum preferred links 2

satellite-fabric-link satellite
To specify an interface as a ICPE inter-chassis link, use the satellite-fabric-link satellite command in the
satellite nV interface configuration mode.
satellite-fabric-link satellite id{slotslot | portsports | network}
Syntax Description
Specifies the satellite id. It is a number in the range 100-239.
satellite id
(Optional) Specifies the slot number. The slot ID consists of two slash-separated numbers,
representing the slot and sub-slot IDs on the satellite device of the node from which to cross-link
ports. This is not supported on single-node satellites.
slot slot id
Specifies the port number. The ports are specified as a range, not necessarily consecutive, of
port IDs to crosslink to this IC Link. The range consists of one or more comma-separated
sub-ranges. Each sub-range can either be a single number, or a hyphen separated consecutive
range (where the left number must be smaller than the right number).
ports ports
(Optional) Specifies a network of satellites.
network
Command Default
The slot defaults to 0/0. The ports default to all available ports.
Command Modes
Satellite Network Virtualization Interface Configuration
Command History
Modification
Release
This command was introduced.
Release 4.2.1
Usage Guidelines
No specific guidelines impact the use of this command.
Task ID
Operation
Task ID
read,
write
ethernet-services
Example
This example shows how to execute the satellite-fabric-link satellite command:
RP/0/RSP0/CPU0:router # configure
RP/0/RSP0/CPU0:router(config)# interface TenGigE0/2/1/0
RP/0/RSP0/CPU0:router(config-int)# ipv4 point-to-point

RP/0/RSP0/CPU0:router(config-int)# nV

show nv satellite crosslink
To display a summary view of which access ports on the satellite are crosslinked to which satellite fabric link,
use the show nv satellite crosslink command in the EXEC mode.
show nv satellite crosslink [brief] [satellite Satellite ID] [interface Interface_Name]
Syntax Description
(Optional) Displays brief information about all the satellite systems.
brief
(Optional) Displays the crosslink information based on the specified satellite
